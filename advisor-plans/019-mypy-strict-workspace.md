## Why this matters

The `mypy.ini` file has `ignore_missing_imports = True` for all 5 workspace packages (`gw2analytics_api.*`, `gw2_analytics.*`, `gw2_core.*`, `gw2_evtc_parser.*`, `gw2_api_client.*`). This means mypy's `--strict` mode is effectively **disabled for the project's own packages**: a broken import path, a type signature change in `gw2_core` that breaks all 5 consumers, or a missing `__init__.py` re-export produces zero mypy errors. The entire type-checking infrastructure is illusory for cross-package calls.

The `mypy_path` (line 26) already includes all 5 workspace source dirs. The comment (lines 58-65) says the overrides exist because "mypy cannot resolve cross-package imports even though pytest can" — but this was never confirmed. The workspace packages are editable-installed via `uv sync`, so mypy *should* resolve them.

## Steps

### Step 1: Remove all 5 `ignore_missing_imports` entries

Edit `mypy.ini` to remove lines 67-79 (the 5 `[mypy-*]` sections for workspace packages). Keep `[mypy-pydantic.*]` (line 55-56) and `[mypy-*.tests.*]` (line 51-52).

**Verify**: `uv run mypy apps/api/src/ libs/gw2_core/src/ libs/gw2_analytics/src/ libs/gw2_evtc_parser/src/ libs/gw2_api_client/src/` — will likely show errors. Record the count and categories.

### Step 2: Fix surfaced type errors

For each error:
- **Import-not-found**: Add missing `__init__.py` re-exports or fix import paths. The workspace packages use `src/` layout — ensure `__init__.py` at `src/gw2_*/__init__.py` re-exports the public surface.
- **Missing type annotation**: Add annotations. Follow the repo convention (Pydantic v2 models use `ClassVar`, `Final` for constants, etc.)
- **Incompatible types**: Fix the actual type mismatch (the bug mypy was hiding).

Strategy: process errors in dependency order (leaf packages first):
1. `libs/gw2_core/src/` — no deps on other workspace packages
2. `libs/gw2_evtc_parser/src/` — depends on gw2_core
3. `libs/gw2_analytics/src/` — depends on gw2_core
4. `libs/gw2_api_client/src/` — depends on httpx only
5. `apps/api/src/` — depends on all 4 libs

For false positives that cannot be resolved cleanly, use inline `# type: ignore[<error-code>]` with a brief comment. Never re-add blanket `ignore_missing_imports`.
