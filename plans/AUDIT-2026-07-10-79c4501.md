## Executive summary

The codebase is genuinely well-engineered where it has been hardened: the webhook secret-at-rest Fernet envelope, the adversarial-input defenses (NUL sanitisation, id dedup, `NUMERIC(20,0)` overflow, DB-mirrored check constraints), the SSRF *create-time* validator, and the CI OpenAPI/health gates are all sound and should not be weakened. The debt is concentrated in three places: **(1) the entire API is unauthenticated** while being published behind a public Caddy vhost, which multiplies several other findings; **(2) the player routes are architecturally inverted** (whole-dataset scan + N+1 per request) with a missing `started_at` index; **(3) release/doc discipline has decoupled from git** (18 untagged "released" versions, a 500-line bloated `[Unreleased]`, archived-but-unapplied plans, docs that contradict each other).

One endpoint (`/compare/timeline`) is **completely broken** on any real dataset (unconditional `KeyError`) — the single most urgent item and a one-line fix.

## CRITICAL

**C1 — `apps/api/src/gw2analytics_api/routes/player_compare.py:183-185` — `/compare/timeline` raises `KeyError` on any non-empty dataset.**
`per_account_contributions: dict[str, list[FightContribution]] = {}` then `per_account_contributions[c.account_name].append(c)` on a plain `dict` — the first contribution for each account hits a missing key. The endpoint returns 500 for any comparison that actually has data; the existing `test_player_compare.py` only exercises the empty-contribution path, so CI stays green. **Fix:** `defaultdict(list)` (or `.setdefault`). **Effort: S.** *(Verified against source during this audit; fix + regression test in progress on a dedicated branch.)*

**C2 — `apps/api/src/gw2analytics_api/routes/uploads.py:152` + `libs/gw2_evtc_parser/src/gw2_evtc_parser/parser.py:624,641` — Unauthenticated memory-exhaustion DoS via unbounded upload read and zip-decompression bomb.**
`raw = file.file.read()` slurps the whole multipart body into RAM with no cap before any check; there is no Starlette body-size middleware (`main.py` adds only CORS) and no `request_body max_size` in the `Caddyfile`. Separately, `read_zevtc_bytes` → `zf.read(names[0])` fully decompresses the first zip entry before the 100 MB `MAX_EVTC_BYTES` guard runs, so a small `.zevtc` inflating to multi-GB OOMs at `zf.read()`. Compounding the risk: parser docstrings (parser.py:158-166, 349-356) assert "The API layer caps at 30 MB (per plan 048)" — **that cap does not exist anywhere in the code**, so the gap reads as already-handled. **Fix:** enforce a streamed/Content-Length cap at the route or a middleware and add `request_body max_size 30MB` to Caddy; bound the zip entry read with a running-size cap (don't trust `ZipInfo.file_size`); delete or make-true the parser comment. **Effort: S (upload cap) / M (zip bomb).**

## HIGH

**H1 — No authentication on the data plane or webhook management (whole API), while published on a public Caddy vhost.**
The only auth is `HTTPBearer` on `GET /api/v1/account` (account.py:49), and that is a *GW2 API key forwarded upstream*, not an app credential. `POST /uploads`, all `/fights` + `/players`, `/webhooks` CRUD, and `/webhooks/dlq/{id}/replay` are fully open. Read this as "any unauthenticated caller can do this" — it is the root multiplier for H2/H3/C2 and the SSRF finding. **Fix:** decide the tenancy model; if internet-facing, put auth in front of the data plane and webhook management. **Effort: M.** *(Severity is HIGH under a single-operator assumption, CRITICAL if genuinely internet-exposed — owner must arbitrate.)*

**H2 — `apps/api/src/gw2analytics_api/workers/webhook_dispatch.py:277` + `webhook_scheduler.py:237` — Webhook SSRF is TOCTOU (DNS-rebind window).**
`_validate_webhook_url` (webhooks.py:91-149) is thorough *at create time* (IPv4+IPv6, private/loopback/link-local/multicast, `getaddrinfo`, fail-closed on DNS error) but the outbound POSTs happen arbitrarily later against the stored URL with **no re-resolution check**. An attacker registers a host resolving to a public IP, then repoints DNS to `169.254.169.254` (cloud IMDS) or an internal host. Blind SSRF (response not returned), and `follow_redirects` defaults to `False` (verified — closes the redirect bypass), but the rebind path is live. **Fix:** re-run the resolved-address block immediately before each POST, or connect to a validated pinned IP with Host/SNI preserved. **Effort: M.**

**H3 — `apps/api/src/gw2analytics_api/routes/players.py:394,483,712` — Unauthenticated PII exposure (GDPR-relevant) + PII in logs.**
All GW2 `account_name`s + per-player damage/healing/role history are world-readable via `/players`, `/players/{account_name}`, `/players/{account_name}/timeline`. Account names are personal data. PII is also logged at INFO/WARNING (services.py:188-196, 234-240, 607-613, 655-662) and DEBUG (parser.py:581-585). **Fix:** gate the data endpoints behind auth (folds into H1); drop/hash `account_name` in log lines. **Effort: M (auth) / S (logging).**

**H4 — `apps/api/src/gw2analytics_api/routes/players.py:113-118,148-168,441-456,548-560,732-743` — Player routes: whole-dataset scan + per-fight N+1.**
`_compute_contributions` loops fights and issues one `SELECT ... WHERE fight_id = ?` per fight (≈ `4 + N` queries per request; ~200 for a 200-fight account) across `list_players`, `get_player`, `get_player_timeline`, and `player_compare`. `get_player`/`get_player_timeline` load **every** `OrmFight` with `selectinload(agents, skills)` then discard all but one account in Python — while the purpose-built `ix_fight_player_summaries_account_fight` index is never used by any route. **Fix:** batch the summary load into one `WHERE fight_id IN (:ids)`; for single-player routes query `WHERE account_name = :acct` (uses the existing index); stop `selectinload`-ing agents/skills on the fast path. **Effort: S (batch) → M (route rework).**

**H5 — `apps/api/src/gw2analytics_api/models.py:122` (migration 0001) — No index on `fights.started_at`.**
Every fight list and all four player routes `ORDER BY started_at DESC` with no supporting index → full-table sort on the hottest ordering column on every call. **Fix:** `CREATE INDEX ix_fights_started_at ON fights (started_at DESC)` (new alembic migration). **Effort: S.**

**H6 — `apps/api/src/gw2analytics_api/routes/fights.py` (all 5 per-fight endpoints) — Same blob re-loaded / re-decompressed / re-parsed up to 5× per fight-page view, no cache.**
`/events`, `/timeline`, `/timeline/players`, `/squads`, `/skills` each call `_load_fight_events` independently; the frontend drill-down fetches them in parallel → up to 5 simultaneous MinIO `get_object` + `gzip.decompress` + full `TypeAdapter` parse of the *same* 60k-event blob. `_load_fight_events` also materialises up to 60k Pydantic objects per request under a ~40-worker threadpool. **Fix:** in-process TTL/LRU cache keyed on `events_blob_uri` (+ parser_version), or fold the drill-down into fewer endpoints. **Effort: M.**

**H7 — `apps/api/src/gw2analytics_api/schemas.py` (whole file) — Pydantic `*Out` mirror schemas are hand-maintained with zero compatibility tests; drift has already materialised twice.**
No test asserts `*Out` schemas stay field-compatible with the `libs/gw2_analytics` row models. Confirmed drifts: `EventBucketOut` (schemas.py:71-87) omits `buff_removal_total` (documented as intentional, but unenforced); `PerFightBreakdownRowOut` (schemas.py:326-355) omits `power_damage`/`condi_damage` that `FightContribution` gained in v0.10.5 — the condi/power split is **silently not surfaced**. **Fix:** `test_schema_mirrors.py` asserting `set(Out.model_fields) - {intentional_drops}` ⊆ `set(LibRow.model_fields)`, with the drops as an explicit allow-list. **Effort: M.**

**H8 — Release/tag discipline decoupled from git (CHANGELOG + tags + docs).**
`git tag` stops at `v0.9.0`; CHANGELOG marks `[0.10.3]` "released 2026-07-10" and README references a "v0.10.2–v0.10.8 cycle" — ~18 versions exist in prose with no git tag (README:12 admits "tags pending operator ceremony"), so the "Latest tag" badge and any tag tooling read v0.9.0. `[Unreleased]` is ~502 lines (CHANGELOG.md:21-523), ~12% of the file, bloated with already-shipped v0.9.0/v0.9.1/v0.10.0/v0.10.1 content that has dedicated versioned sections elsewhere; section order is non-monotonic (`[0.10.3]` sits *above* `[Unreleased]`). **Fix:** tag the shipped releases; drain `[Unreleased]` into versioned sections; reorder descending. **Effort: M.**

**H9 — `docs/v0.8.0-backend-design.md` — Design doc materially stale AND its fix-plan (062) was archived without being applied.**
Confirmed drifts still present: webhook User-Agent frozen at `0.8.0`; `webhook_deliveries` schema missing `next_attempt_at`/`payload` (migration 0007) and showing `delivered_at NOT NULL` though the ORM made it nullable; `secret` shown as plaintext TEXT despite the Fernet envelope; replay endpoint (shipped) listed as "future"; doc lists 3 webhook endpoints, code has 5. Plan 062 (which prescribed these exact fixes) is in `plans/archive/` — i.e. marked done — but the fixes were never applied. **Fix:** apply plan 062's edits (or banner-freeze the doc) AND sweep `plans/archive/` for other archived-but-unapplied plans. **Effort: M.**

## MEDIUM

**M1 — `apps/api/src/gw2analytics_api/main.py:208` — `FastApiMCP(app).mount()` republishes the entire (unauthenticated) API as an MCP tool surface.** Inherits every H1 gap over a second protocol. **Fix:** gate or disable in prod. **Effort: S.**

**M2 — `apps/api/src/gw2analytics_api/config.py:71` + `main.py:172-177` — CORS defaults to wildcard `["*"]`.** `allow_credentials` is `False` (so not a credentialed-CSRF hole), but a prod deploy that forgets `CORS_ALLOWED_ORIGINS` ships wide-open. **Fix:** default to empty; require explicit origins. **Effort: S.**

**M3 — No rate limiting anywhere (app or Caddy).** Unauth + no limits + CPU/memory-heavy parse per upload + SSRF fan-out via webhook creation = cheap DoS. **Fix:** per-IP limits at Caddy/app, especially `POST /uploads` and `POST /webhooks`. **Effort: M.**

**M4 — `apps/api/src/gw2analytics_api/routes/uploads.py:146-207` — Sync blocking I/O on the event loop in an `async` handler.** `file.file.read()` and `put_zevtc(...)` (MinIO PUT) run directly on the loop; the parse itself is correctly offloaded but the read + PUT are not. **Fix:** `asyncio.to_thread` or `await file.read()`. **Effort: S.**

**M5 — Unbounded per-bucket / per-series payloads on long fights.** `/fights/{id}/timeline` `points`, `/events` `event_windows`, and especially `/fights/{id}/timeline/players` (`series`) are uncapped: a 60-min WvW fight at `window_s=1` → ~3600 buckets × dozens of players ≈ 144k zero-filled points in one response — the same blow-up class hotfix #12 fixed for rollups, via long-fight × small-window instead of garbage IDs. **Fix:** cap bucket count or raise the `window_s` floor for long fights. **Effort: S.**

**M6 — Time-bucketing skeleton triplicated across `event_window.py:98-208`, `per_fight_timeline.py:152-268`, `per_player_timeline.py:188-329`.** `time_ms // window_ms` indexing, zero-fill, `_MIN_WINDOW_S` guard, 3-way `isinstance` dispatch, and the `pairwise` contiguity invariant are copy-pasted; the code has flagged "extract a shared `_bucket_by_window_ms`" since v0.8.x and a third clone was added anyway. The desc-sort + sum-preservation invariant checks are also pasted across 6+ aggregators. **Fix:** extract `_bucketing.py` + `_invariants.py` helpers. **Effort: M.** *(This is the natural next refactor after the merged 084/085/116.)*

**M7 — Day-bucketing duplicated across the lib/app boundary and already drifted.** `_combine_day_midnight` + the day-bucket loop live in both `cross_account_timeline.py:239-297` (typed `datetime`) and `routes/players.py:620-709` (still `Any -> Any`). **Fix:** move to a shared `gw2_analytics.time_buckets` helper; both import it. Closes the residual half of advisor-plan 002. **Effort: M.**

**M8 — Densest modules lack dedicated unit tests.** `routes/fights.py` (802 LOC) has no `test_fights.py` (only the per-target helper is unit-tested); `services.py` (708 LOC, `_persist_player_summaries`) and `crypto.py` have only indirect e2e coverage. **Fix:** per-endpoint contract tests for fights (404/502/empty branches), `test_services_persist.py` (fast/slow parity), `test_crypto.py` (round-trip + wrong-key). **Effort: L.**

**M9 — Schema-drift guard is a boot-time version-pointer check, not a CI structural gate.** `schema_guard.py:99` only compares the alembic version pointer; nothing runs `alembic --autogenerate` in CI to catch ORM-vs-migration structural drift (e.g. `String(64)` vs `String(128)`). **Fix:** add a CI step running autogenerate against the Postgres service and failing on non-empty diff. **Effort: M.**

**M10 — CI has no dependency audit / secrets scan / coverage gate, and README claims one that doesn't exist.** README:12 claims "CI security hardening (pip-audit + pnpm audit)" — `ci.yml` contains neither (nor gitleaks/CodeQL/coverage). **Fix:** add `pip-audit` + `pnpm audit` (non-blocking to start) + a secrets scanner; or remove the false claim. **Effort: M.**

**M11 — Duplicate plan numbers.** `plans/135`–`plans/138` each exist twice with different slugs, breaking the "numeric prefix = unique ID" convention. **Fix:** renumber the collisions. **Effort: S.**

**M12 — Docs disagree on "current state."** ROADMAP.md (self-declared source of truth) says v0.8.9 / 303 tests; README says v0.9.1 tag / 1,150+ tests; CHANGELOG top is 0.10.3. Roadmap also lists cross-account comparison (shipped, `player_compare.py:89`), CSV-injection guard (shipped), and secret-at-rest (shipped) as live candidates — its actionable shortlist is effectively empty. **Fix:** reconcile against a real test run; refresh ROADMAP current-state; delete shipped items. **Effort: S.**

## Verified-sound — DO NOT weaken

These were checked and are correct; the complexity protects a real invariant:

- **Webhook secret-at-rest**: Fernet envelope fully implemented (`crypto.py`, `models.py:337` stores only ciphertext, no plaintext column; KEK is a validated 32-byte `SecretStr`, fail-fast at startup, no hardcoded prod key, per-row corrupt-ciphertext tolerance). Closes the CWE-256 the CHANGELOG claims.
- **SSRF create-time validator**: complete classifier (IPv4+IPv6, literal-vs-hostname, `getaddrinfo`, private/loopback/link-local/multicast/ULA, fail-closed DNS, scheme restriction, fail-safe env escape hatch). Only the send-time TOCTOU (H2) is open.
- **Adversarial-input defenses**: NUL strip + 128-char truncation at every ORM write boundary, agent_id + skill_id dedup, `agent_id NUMERIC(20,0)` for uint64, `MAX_AGENTS`/`MAX_SKILLS`/`MAX_SKILL_NAME_BYTES` bounds, DB `CheckConstraint`s mirroring the service guards.
- **No SQL injection** (ORM/parameterised throughout), **no path traversal** in blob URIs (derived from server-side SHA-256), **httpx does not follow redirects**.
- **CI OpenAPI drift gate** and **health gate** are real, in-process, and meaningful; `uv sync --frozen` + `pnpm --frozen-lockfile`; Postgres + Vitest + Playwright + PR-only visual regression all run in CI; dependabot config is well-reasoned.
- **`libs/gw2_core` domain tests** and **`libs/gw2_analytics` aggregator tests** are solid coverage (advisor-plans 001/003/005/006 are DONE).
