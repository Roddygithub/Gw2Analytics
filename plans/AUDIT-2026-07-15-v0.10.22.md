## Executive summary

The v0.10.22 mimo-half cycle shipped the FIRST implementation of the `Skill build analyser` design-doc item from `docs/v0.8.0-web-design.md` §6 (the per-player skill attribution view on the `/fights/[id]` drill-down page). The cycle was provision-drafted at `plans/RELEASE-v0.10.22.md` (this file's sibling) and the design intent was pre-listed in `docs/ROADMAP.md` §1 v1.0 candidates (the "Skill build analyser" entry sourced from §6, marked M-effort). The cycle's scope: 3 backend files + 4 backend pytest cases + 6 frontend lib/api types + 2 frontend lib/api functions + 2 new Client Components + 1 page-integration edit + 2 new vitest files (17 cases) + 1 page-test extension (4 cases) + 1 mock-server extension (2 endpoints) + 1 Playwright extension (3 cases). 6 atomic code+tests commits + 3 docs commits = 9 atomic events on `main`. ZERO production-code regression on the K-cluster (Tour 4 does NOT touch `apps/api/tests/conftest.py`; the v0.10.20 PARTIAL-FIX inherited surface is untouched). ZERO regression on the existing 5 per-fight endpoints (the new artisan route + 2 new fetcher functions are additive). ZERO vitest/playwright/pytest test regression on the pre-v0.10.22 surface (the new mock-server entries are additive; the new component mocks in setup.ts are additive; the new page-test cases are additive).

**Shipped:** 9 atomic commits on `main` + annotated tag `v0.10.22` + `gh release create`.

**Forward-blockers carried to v0.11.0 / v0.11.X** (3 items):
1. **Parser-layer equipped-skill extraction** — the `equipped_skill_ids` field on `PlayerSkillLoadoutOut` is the V1 STUB (always `[]`); the parser-side extractor for equipped-skill IDs from the EVTC binary is deferred to v0.11.0.
2. **Case-insensitive `?account=` URL match** — the page's agent lookup is STRICT byte-for-byte account_name equality; a future case-insensitive contract is a v0.11.X followup.
3. **Source-agent-id-zero guard** — the per-player event pre-filter accepts `source_agent_id == 0` events (the parser's "global no-attribution" sentinel); a future v0.11.X followup adds the canonical drop.
