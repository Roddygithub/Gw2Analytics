## §2 Why MINIMAL scope

The widening was scoped to ONLY `PlayerReadoutOut.account_name` even though the schema search surfaced 8 occurrences across 5 types. The choice rationale documented inline in the schema + aggregator commit messages:

- `AgentOut`: already `account_name?: string | null` in TS (line 828 in `schema.d.ts`); widen came in lockstep with the Python `str | None = None` already present
- `PlayerListRowOut` + `PlayerProfileOut` + `PerPlayerTimelineSeriesOut` + `CrossAccountTimelineSeries`: keep `account_name: str` non-Optional because the cross-account data source (the backfill at `apps/api/src/gw2analytics_api/backfill.py:172-179`) filters on `if a.is_player and a.account_name` — the data-pipeline contract guarantees non-null so the non-Optional schema is sound

The minimal-scope decision avoids cascading wire-contract widenings into paths that have NO silent-failure mode. The one follow-up cycle scoped for v0.11.0 (per the `TODO(v0.11.0)` note in the schema comment): if the backfill softens its non-null guarantee, widen those 4 types in v0.11.0 with the same atomic commit pattern.

## §4 Migration impact + adoption checklist

For consumers of the `GET /api/v1/fights/{fight_id}/readout` endpoint:

1. **TypeScript type re-gen**: regenerate `schema.d.ts` from the OpenAPI dump to confirm the `account_name: string | null` widening survives auto-gen. The manual edit + `# noqa: wire-followup` marker (commit 5725423 fix #4) is in place; the marker must be reconciled manually if a future regeneration overwrites it.
2. **Frontend rendering**: code paths that previously assumed `account_name: string` and used it as a React key (style.display, `${...}` template literal, etc.) must add a `?? "fallback"` at the consumer site. The previous crossAccount fix (commit 5725423 fix #1) closed the most-egregious case in `CrossAccountTimelineChart.tsx`.
3. **TypeScript nullability tests**: any consumer whose assertion was `expect(row.account_name).toEqual("expected_string")` for a row whose underlying OrmFightAgent had `account_name: null` will need an `is None` branch. The combat-readout TestSuite (commit 5725423 fix #2) provides the None-path pin for apps/api tests; web equivalents are F17 cycle scope.
