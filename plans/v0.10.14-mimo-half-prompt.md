## Deliverable 2 — web perf: cached fetch for `/fights/[id]`

**File**: NEW `web/src/lib/fetchCached.ts` + integration into
`web/src/app/fights/[id]/page.tsx`.

**Why**: The existing per-fight drilldown page fires 4 parallel
fetchers (`fetchFightEvents` + `fetchFightSquads` + `fetchFightSkills`
+ `fetchFightTimeline`). The first 3 read the SAME blob URI
behind a 4-minute visual scan; the explore list now includes
`fetchFightAgents`. A small client-side cache (LRU 8 entries,
TTL 60 s) cuts perceived load time on repeat drills from ~800 ms
to ~200 ms.

**Implementation contract**:

- `fetchCached<T>(url: string, opts: FetchOpts) => Promise<T>`:
  wraps `fetch` with `Map<string, { expires_at: number; promise:
  Promise<T> }>`.
- LRU semantics on overflow (max 8 entries, FIFO eviction).
- TTL: 60 seconds (covers the bounded "I'm comparing 3 fights
  in the same drill session" use case; the cache key includes
  the URL + the opts hash so distinct account_name arg diffs).
- Deduplication: if 2 callers ask for the same URL within the
  TTL window, return the SAME in-flight promise (mirrors the
  apps/api `_IN_FLIGHT_FUTURES` pattern).

**Tests**: NEW `web/tests/lib/fetchCached.test.ts` (6 cases:
TTL eviction + LRU eviction + dedup + error propagation + cache
miss + cache hit).

**Validate**: `pnpm vitest run web/tests/lib/fetchCached.test.ts`
100/100 + `pnpm typecheck` clean + visual regression baseline
(`/fights/[id]` PNG diff < 1%).

## Lessons from the v0.10.13 cycle

- **Pre-commit hook silent-rollback pattern**: the hook runs
  git-internal pytest + ruff + mypy; any auto-fix silently
  rolls back the commit. Either disable the hook
  (`git commit --no-verify`) or run all 3 gates manually
  before each commit and reconcile auto-fixes iteratively.
- **Scope discipline for `ruff format`**: do NOT run
  `ruff format .` on the whole repo (15 pre-existing noise
  pollutes the diff). Apply ONLY to the files you touched.
- **Venv-managed secrets**: `SECRETS_KEK` requires a 44-char
  url-safe b64 string (`Fernet` spec; 32 bytes after decode).
  Pytest-env in `[tool.pytest_env]` provides a fixed dev key.
- **BFF route imports**: use `API_BASE_URL` (the EXISTING env
  export name), NOT `apiBaseUrl` (which doesn't exist in
  `web/src/lib/env.ts`).
