### D4 — `fetchCached` LRU isolation regression-pin test (deferred v0.10.16 D4)

**Scope:** Land the 5-behavior pin test that the v0.10.16 D4 deliverable scoped but did not execute. Per the v0.10.16 brief's MUST-FIX D4 contract, this test MUST contain **≥5 sub-cases** (one per pinned behavior).

**File:** `web/tests/lib/fetchCached-isolation.test.ts` NEW.

**The 5+ pinned behaviors:**
1. **TTL hit** — 2nd same-URL call within 60s returns the same cached value (zero new network round-trips).
2. **TTL expiry** — same-URL call after 60s+ re-fetches (1 new round-trip).
3. **Dedup** — N overlapping same-URL calls collapse to 1 network round-trip (in-flight dedup before TTL).
4. **No-cache-on-error** — a fetcher rejection does NOT cache the rejection (retry gets a fresh attempt).
5. **LRU cap eviction at maxsize=8** — the 9th distinct URL evicts the oldest (the `maxsize=8` cap is a hard memory bound).
6. (Optional) **Concurrent dedup** — N truly parallel same-URL calls (Promise.all) yield 1 round-trip + N-1 awaited results.

**D4 done criteria:** `cd web && pnpm vitest run tests/lib/fetchCached-isolation.test.ts` reports all 5+ sub-cases passing.

### D5 — Replay UI + cycle hygiene anti-regression pin

**Scope:** Land a single test that pins both deliverables D1-D3 against future regressions. This consolidates the "no future cycle can break these" guarantee.

**File:** `web/tests/lib/replay-ui-fetch-cached.test.ts` NEW OR a section appended to `fetchCached-isolation.test.ts`.

**Pinned behaviors:**
- The Replay UI's `replayFetcher` uses `fetchCached` (NOT a direct `fetch`). This pins the D1 contract.
- The fetch is correctly dedup'd across overlapping Replay play/pause toggles (verifies the D1 ↔ v0.10.14 D2 contract).
- After the Replay tab unmounts + remounts, the second mount reuses the cached entry (TTL hit, no re-fetch).
- The Replay UI correctly handles "events blob 404" from the gateway (the error card shows + the LRU does not cache the error).

**D5 done criteria:** vitest reports the D5 test passing + the prior D1-D4 tests all still pass.

## Failure modes (NAKE escalation triggers)

- **D1 introduces >500 LoC of new web component code.** This suggests the design isn't reusing `fetchCached` + the existing tab strip. Defer to a fresh plan revision.
- **D3 introduces a new failure** (net test count decreases vs pre-D3) — likely an unexpected coupling.
- **A fix requires >50 LoC of production-code changes** — the D1 classification was wrong; needs more diagnostic work first.
- **The fix would undo a v0.10.15 or v0.10.16 hygiene contract.**
- **Test runtime increases by >2× vs pre-fix** — likely a bare `sleep` rather than a real fix.
- **D3 over-defers >4 of 9 failures into bin "deferred"** — cycle scope needs re-evaluation.
- **D4 misses any of the 5 pinned behaviors** — reviewer's MUST-FIX MUST inline-fix.
- **D1 Replay UI breaks the existing `/fights/[id]` page** (the existing tab interactions must continue to work; D1 is purely additive).
