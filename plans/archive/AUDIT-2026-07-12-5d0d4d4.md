## Rejected alternatives (audit-time)

- **Resolve O4 via error-boundary components**: Next.js `error.tsx`
  boundaries only catch the deep render, not the per-fetch failure
  in a Server Component's `Promise.allSettled`. The boundary approach
  loses per-section granularity. **Rejected** — keep the
  per-section "missing sections" UI affordance.
- **Resolve O5 by skipping the pre-existing tests with `pytest.skip`/`vitest.skip`**:
  Hygiene-degrading; the FAIL signals real per-cycle regression risk.
  **Rejected** — fix the underlying assertions.
- **Resolve F20 by tree-shaking ag-grid**: ag-grid-community does
  NOT tree-shake (the `AllCommunityModule` import is eagerly
  evaluated for backwards-compat grid setup). Even moving to a
  per-module import path keeps the styles bundled. **Rejected** —
  defer until a deferred-import path is published upstream.
- **Resolve F8 by further splitting `services.py`**: the v0.10.4
  split IS the resolution; further splitting is YAGNI. **Rejected**
  — log as PARTIALLY RESOLVED.
- **Add a per-cycle audit checkpoint (audit every cycle instead of
  per-version)**: too much maintenance tax; per-major-version audits
  are sufficient. **Rejected** — keep per-major-version cadence.
