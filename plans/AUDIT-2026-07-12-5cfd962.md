## Considered and rejected

- **Single combined CI-gate-buster commit covering ruff + mypy + openapi**: tempting (one PR to unblock all 3 gates at once). Rejected: each gate is independent at the test fixture level; combining them conflates 3 separate invariants (lint + type + schema-drift) and makes any one harder to revert if a regression lands in CI.
- **Skip the OpenAPI regen and accept the drift gate failure**: tempting (it's just 1 file + 1 workflow step). Rejected: the v0.9.1 plan-008 work was deliberate — the gate is the operator's defense against silent schema drift. Bypassing it would unwind the entire v0.9.1 hardening posture.
- **Auto-fix the 19 mypy errors via `--strict-partial` or `ignore_errors`**: tempting (silence the gate without manual work). Rejected: that would unwind plan 019 (mypy strict) which was the explicit deliverable of the 2026-07-11 audit. The mypy errors must be addressed in source.
- **Re-derive the test count from `git log -p` (extract test counts from PR descriptions)**: tempting (no automation needed). Rejected: too fragile (PR descriptions are not a stable contract). The right path is the CI-artifact approach in plan 030.
- **Revert PR 3.2 (FightsPageOut) to unblock the OpenAPI gate quickly**: rejected. The wrapper is the architecturally correct shape; reverting it would unwind PR 3.2 (the deferred closing piece of plan 021). The schema regen is the right path.
- **Aggressively thin `routes/fights/__init__.py` below 600 LoC**: rejected. The 703 LoC is mostly native FastAPI route decorators + Pydantic v2 schema construction. Splitting into controller objects exceeds idiomatic FastAPI norms and adds indirection without reducing LoC meaningfully.
